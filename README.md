# Prompt Eval

> 一个轻量级的 Prompt 评测与迭代工具，帮助开发者系统化地评估和优化 AI Prompt。

![Prompt Eval Demo](prompt-eval-demo-25s.mp4)

## 快速开始

直接在浏览器中打开 `prompt-eval-studio.html` 即可使用，无需安装任何依赖。

```bash
open prompt-eval-studio.html
```

## 核心流程

Prompt Eval 遵循**六步迭代流程**，帮助开发者从定义任务到持续优化：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐         │
│   │  1.定义  │───▶│ 2.构建   │───▶│  3.LLM   │───▶│  4.Code  │         │
│   │   任务   │    │ 测试集   │    │  Grader  │    │  Grader  │         │
│   └──────────┘    └──────────┘    └──────────┘    └──────────┘         │
│        │                                               │                │
│        │                                               │                │
│        ▼                                               ▼                │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐         │
│   │          │◀───│ 6.迭代   │◀───│ 5.运行   │    │          │         │
│   │          │    │  对比    │    │  评测    │    │          │         │
│   └──────────┘    └──────────┘    └──────────┘    └──────────┘         │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 六步流程详解

1. **定义任务** — 明确 AI 需要完成的任务类型和预期输出格式
2. **构建测试集** — 创建多样化的测试用例，覆盖典型场景和边界情况
3. **配置 LLM Grader** — 设置语义评分标准，定义"好回答"的标准
4. **配置 Code Grader** — 添加规则校验，如格式检查、关键词匹配等
5. **运行评测** — 系统自动运行测试，生成详细的评测报告
6. **迭代对比** — 根据失败用例优化 Prompt，v1/v2 对比验证改进效果

## 功能特性

### 🎯 双路评分系统

- **LLM Grader**: 基于语义理解的质量评分（权重可配置，默认 70%）
- **Code Grader**: 基于规则的自动化校验（权重可配置，默认 30%）

### 📊 智能评测报告

- 总体评分和通过率统计
- 单用例详细分析（得分、耗时、Token 消耗）
- 失败用例归因分析
- v1/v2 版本对比视图

### 🤖 AI 辅助功能

- **自动生成测试用例**: 基于任务描述智能生成测试集
- **自动生成 Rubric**: 自动创建评分标准和评分维度
- **智能权重建议**: 根据任务类型推荐 LLM/Code 权重配比

### 📈 迭代优化

- 一键生成 v2 Prompt 建议
- 并排对比 v1/v2 评测结果
- 失败用例聚类分析

## 使用示例

### 场景 1: 客服机器人评测

```javascript
// 任务定义
任务: 回答客户关于订单状态的咨询

// 测试用例示例
{
  "input": "我的订单 #12345 什么时候能到？",
  "ideal": "您好，订单 #12345 预计将在 2-3 个工作日内送达。您可以通过以下链接实时追踪物流状态..."
}

// LLM Grader 评分维度
- 准确性: 是否提供了正确的订单信息
- 完整性: 是否包含预计送达时间和追踪方式
- 语气: 是否保持友好、专业的客服语气
```

### 场景 2: 代码生成评测

```javascript
// 任务定义
任务: 将 AWS CLI 命令转换为 AWS SDK (Python boto3) 代码

// Code Grader 规则示例
- 语法检查: 生成的代码是否可编译/执行
- 参数映射: CLI 参数是否正确映射到 SDK 参数
- 最佳实践: 是否遵循官方推荐的使用模式
```

## 技术说明

- **纯前端实现**: HTML + CSS + JavaScript，无需后端服务
- **API 兼容**: 支持 OpenAI 兼容的 API 格式（如 OpenRouter）
- **数据本地存储**: 评测数据保存在浏览器本地，保护隐私

## 配置说明

在页面右上角输入 API Key 后即可开始使用。支持以下配置：

- **API Endpoint**: 默认 `https://openrouter.ai/api/v1/chat/completions`
- **模型选择**: 支持 GPT-4、Claude 等 OpenAI 兼容模型
- **评分权重**: LLM Grader / Code Grader 权重配比（默认 70%/30%）

## 最佳实践

1. **测试集设计**: 覆盖典型场景 + 边界情况 + 对抗样本
2. **评分标准**: LLM Grader 关注语义质量，Code Grader 关注格式正确
3. **迭代策略**: 先分析失败模式，再针对性优化 Prompt
4. **版本对比**: 使用 v1/v2 对比验证改进效果，避免回归
